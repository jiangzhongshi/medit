Download the application at the following address: http://www.ljll.math.upmc.fr/~frey/logiciels/medit2.html

The file decompresses automatically (if it does not, you must open it in StuffIt Expander to unpack it). Then, place the Medit application in your Applications folder.

If you want to use Medit in a terminal, you just need to create a script that contains the following information:
#! / Bin / sh
/Applications/medit.app/Contents/MacOS/medit "$ @"
Name it Medit And place it in a place seen by your PATH . Therefore, when you type the Medit Command in a terminal window, the application will launch automatically.

Warning: Medit was created for a machine with a three-button mouse, so to have access to all the features, it is recommended to connect one to your Mac. However, if you have only one mouse at a button, you can get the right button by typing ctrl + click and the left button by typing alt + click

2 Getting Started and File Format

2 . 1 File Format


2 . 1 . 1 Mesh Format

Back to overview

Medit allows mesh meshes (binary or ASCII) to be displayed . This format is composed of a single file ( xx.mesh or xx.meshb ) that contains all the information needed to describe the mesh. It is organized into a series of fields, identified by a keyword. The comment lines begin with the character and the "blank", the line (or <CR> ) and the tabs are considered as separators.

The file must start with the following 2 lines:
MeshVersionFormatted 1
Dimension 3
NB : the dimension can also be equal to 2.

The fields corresponding to the geometry must then be defined: the vertices of the mesh as well as these elements (triangles, quadrilaterals, tetrahedrons, hexahedrons, etc.). These entities are defined as follows:
NB : The numbers of the entities begin (and not as in some language)


Vertices ... with the number of nodes of the mesh, the coordinates of the point and its reference. NB : In dimension 2, we define only Triangles ... with the number of triangles of the mesh, the number of the vertex of the triangle and its reference. Note : Each reference corresponds to a color in Medit Quadrilaterals ... with the number of quadrangles of the mesh, the number of the vertex of the quadrangle and its reference. NB : Each reference corresponds to a color in Medit Tetrahedra ... with the number of meshed tetrahedrons, the number of the tetrahedron 's top and its reference. NB : Each reference corresponds to a color in Medit Hexaedra ... with the number of hexahedra of the mesh, The number of the hexahhedron vertex and its reference. NB : Each reference corresponds to a color in Medit Edges ... with the number of edges of the mesh, the number of the two vertices that make up the edge and its reference.


















































Back to overview

It is then possible to define the singularities of the mesh and its constrained entities.
The corners correspond to a point of discontinuity and by analogy, ridges can be defined as edges which have a discontinuity between their two adjacent faces. The constrained entities ( RequiredVertices or RequiredEdges ) correspond to entities that are to be preserved in the case of a rearrangement. Fields will be defined as follows:


Corners ... with the number of corners and a vertex number. RequiredVertices ... with the number of constrained points and a vertex number. Ridges ... with the number of "rigde" and an edge number. RequiredEdges ... with the number of edges constrained and an edge number.





























Back to overview

Then the file can contain information about the normals and the tangents to the surface. Normals and tangents are given as a list of vectors. Normal in ( NormalAtVertices are specified using a number of vertices and an index corresponding to a normal vector. The normal triangles, ( NormalAtTriangleVertices ) correspond to a triangle number, a point index in the triangle and a vector index . normal at this point even the field was ( NormalAtQuadrilateralVertices ) above the definition of these different fields.


normals ... with the number of normal and components of normal Tangents ... With the number of tangents and the components of the tangent NormalAtVertices ... with the number of normals at the vertices and the number of the vertex and the number of the normal at this vertex. NormalAtTriangleVertices ... with the number of normals at the vertices of the triangles, the number of the triangle, the number of the vertex in the triangle and the number of the normal at that vertex. NormalAtQuadrilateralsVertices ... with the number of normals at the vertices of the quadrilaterals, the number of the quadrilateral, The number of the vertex in this quadrangle and the number of the normal in this vertex. NormalAtEdges ... with the number of normals at the vertices of the edges, the number of the edge, the number of the vertex of the edge and the number of the normal in this vertex.













































The file must end with the keyword End .

2 . 2 Start Medit

We assume here that you have a mesh (for example: toto.mesh) that you want to view. To start Medit , simply type medit toto (or medit toto.mesh ) in a terminal window. A window containing the mesh will open.

Warning : if you do not specify the extension (.mesh (b)) of your file, Medit first looks for a binary file and then if it does not find it it opens the ASCII file. In other words, if you have in the same directory the files toto.mesh and toto.meshb and you type medit toto , you will see the file toto.meshb .

  • Medit -h : order summary
  • Medit -a : to start an animation, see 8
  • Medit -d : debug mode, allows to have more info
  • Medit -fs : fullscreen mode, to have the image in full screen
  • Medit -i : Prints OpenGL Configuration Information

5 Cutting plane

    By default, this is the plane of equationwiththe abscissa of the barycentre of the mesh.
    The equation of the section plane is written at the bottom of the window.
  • Edit clip (F2): allows to move this plane
    With the left button of the mouse the plane turns
    With the middle button the plane of movement parallel to its axis.
  • Freeze clip (F3): allows to move the object around the cutting plane
  • Inverse orient : inverses the orientation of the plane
  • Toggle capping : makes a flat cut plane
  • Toggle plane : removes the "trace" from the cutting plane
  • Toggle VClip (F4): mask the tetrahedra

7 Saving images

Medit provides a simple way to save views. However, it should be noted that only bitmap images can be exported (ie screenshots). Two export formats are possible:
  • Ppm format : to save a ppm (portable pixmap) view, go to the menu ( ctrl + click), in the FILE category and click on Hardcopy PPM . Note that the size of the image (in pixels) is the size of the current window, resulting in the size of the exported image not exceeding the resolution of the screen.

  • Bitmap format postscript : Medit offers three options to create a postscript bitmap image of the window that you view: color, gray or black white. To create an image, go to the menu ( ctrl + click), in the FILE category and click on Hardcopy EPS .
    By default, the size of the postscript file depends on the size of the graphics window. It is possible to control the size of the postscript image (in cm) and its density (in dot per inch), for this, you must define these data in the DEFAULT.medit (see 10 ).

8 Animation

Here we consider the case where a meshing sequence (or solutions on a mesh) has been computed and stored. Medit allows the user to visualize the result of the mesh sequence very easily. The aim being to fix a view on the first mesh of the sequence and then to visualize all the meshes of the sequence successively.

To do this, it is first necessary that the mesh files have a name of the form: toto.XXX.mesh (for example for a sequence of 121 meshes, one will have files ranging from toto.001.mesh to foo. 121.mesh). In other words, the names of the files are divided into 3 parts (separated by "."): A character string common to all the files, 3 digits designating the mesh numbers in the sequence, and the mesh (b) extension.
Then you must run Medit in the animation mode: medit toto -a 1 121
The syntax of the command is as follows: you put your mesh name followed by -a followed by the number of the first mesh and the number of the last mesh.
A graphical window s' Then opens with your first mesh, you can then choose the view, colors etc. etc .. that will apply to your sequence. Then click on the menu ( ctrl + click) in the Animation category on Play Sequence to start the animation.

9 Viewing two meshes simultaneously

If you want to compare two meshes or two solutions, you can open them simultaneously in Medit . To do this, just type medit toto.mesh tata.mesh (if toto.mesh and tata.mesh are the two meshes concerned). In this case, two graphic windows will open. If one wants to compare the two meshes, it is interesting to "bind" them (that is to say that when one moves, the other will make the same displacements). To do this, go to one of the two windows and in the menu ( ctrl + click) in the category view click on copy ( alt + c ). Then, Another window and click on link ( alt + l ). From then on, when you move the first mesh, the second mesh will automatically follow. To "untie" the second mesh, just click an unlink u .

Instead of linking the two views, you can simply copy a view ( copy or alt + c ) and paste it into another window ( paste or alt + p ). In this case, the second mesh will remain independent of the first.

10 The file "DEFAULT.medit"

In addition to the option menu (which appears by pressing ctrl + click ), it is possible to define certain parameters in a text file. You can therefore define a file toto.medit associated with the mesh toto.mesh that you want to visualize. If you do not set this file Medit searches the directory in which your mesh file DEFAULT.medit (he reads if found).
Note : All parameters defined in this file have default values.

This file consists of one or more fields in the following list:

  • BackgroundColor rgb where r, g and b are numbers betweenand.
    This allows you to give thebackgroundcolor of the image (by default thebackgroundis black soif you setthebackgroundwill be white.
  • LineColor rgb
    This is used to define the color of the edges of the mesh (defaultis white).
  • SunPosition xyz
    This allows you to specify the position of the light source (default.
  • WindowSize wh where w and h are 2 integers.
    This makes it possible to determine the size of the window (w = width, h = height) in pixels.
  • RenderMode str where str one of the following keywords: hidden, fill, colorshading, shading
    This specifies the "rendering mode"
  • Palette c1 c2 c3 c4 c5 where theyare real numbers
    This allows to set the values ​​of the palette when visualizing a solution on the mesh. This pallet is also used for the calculation of isolines and isosurfaces.
  • Postscript em dpi k str
    This is used when creating a postscript image: em define the size of the image in cm, dpi is the desired resolution (by defaultin dot per inch ), k is a coefficient used 'It is not null) for ??? And in the current version str is not used.
  • Time t dt nb
    This is used to activate particle visualization.
    T denotes the final temp, dt the time step and nb the number of consecutive particles.




In this file, comment lines begin with the character , and can only exist between the definition of two fields (see example below).
